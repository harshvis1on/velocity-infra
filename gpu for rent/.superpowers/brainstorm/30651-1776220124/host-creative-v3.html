Design Direction — V3

Typography Fix

  • Instrument Serif ONLY for hero headlines and section titles — the editorial anchor
  • Outfit (weight 300–800) for EVERYTHING else — body, labels, data, stats, code snippets
  • NO monospace font anywhere. Numbers use Outfit 700–800 with tight letter-spacing instead
  • One serif + one sans = cohesive. No font clash.

AI-Generated Images (5 needed)

  • Hero: Abstract GPU chip as monumental architecture — silicon cathedral, dark cinematic lighting
  • Yield cards: Individual GPU renders (RTX 4090, A100, H100, L40S, A6000) — product-shot style, dramatic lighting on dark
  • Thesis 1: Stylized exponential demand curve — editorial data visualization
  • Thesis 2: Server rack rows with one glowing green — the "activated" GPU
  • All images: dark palette, accent green, editorial mood, NOT generic stock

Layout Principles (from design skills)

  • Hero is SPLIT not centered — asymmetric, text-left / image-right
  • Yield section breaks the grid — horizontally scrollable card track
  • Thesis uses editorial zigzag — alternating text/image sides
  • Steps are a numbered list, NOT a 3-card row (anti-3-column-card pattern)
  • min-h-[100dvh] not h-screen for mobile stability
  • No emojis anywhere — tier badges are clean text only

Animation Plan

  • Hero: text stagger-reveal left, image fade-in + subtle parallax right
  • Stat numbers: countUp animation on scroll (300% types up from 0)
  • Yield cards: stagger entrance, spring physics on hover
  • Thesis visuals: parallax offset — image moves at 0.8x scroll speed
  • Steps: sequential reveal — each step fades in as previous completes
  • Tier bar: animated fill/glow that sweeps left to right on scroll
  • All transitions: cubic-bezier(0.16, 1, 0.3, 1) — no linear easing

Performance (fixing slow Vercel load)

  • Replace framer-motion with CSS animations for landing pages — drops ~40KB from bundle
  • Use Intersection Observer for scroll triggers instead of scroll listeners
  • Next.js Image component with priority for above-fold, lazy for below
  • Font subsetting — only load needed weights
  • Dynamic import for heavy components (HeroTerminal, etc.)
